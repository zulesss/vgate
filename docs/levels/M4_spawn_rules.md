## §1 Selection Rule

**Метод: weighted random с anti-repeat защитой.**

Round-robin отклонён. Причины: предсказуем (игрок выучивает цикл за 2 минуты), не адаптируется под позицию игрока, создаёт периодические "безопасные" окна между активациями точки.

**Базовые веса** (сумма = 10):
- S1 (Юг, за спиной игрока): 3
- S2 (Север, фронт): 3
- S3 (Запад): 2
- S4 (Восток): 2

S1/S2 получают больший вес намеренно: S1 = угроза с тыла (tension-creator по M3_identity §3), S2 = дальний стрелок с прямой sightline на всю арену. Фланги S3/S4 активируются реже, но достаточно для кросс-fire давления.

**Anti-repeat: точка не может активироваться дважды подряд.** При выборе точки, совпавшей с предыдущей — перевыбор (один reroll, потом force-следующая по весу). Это ломает паттерн "зона горит" когда три врага подряд идут из S1 и игрок читает его как единственную активную точку.

**Player-position bias (опционально, для систем-дизайнера на M6):** Не вводить в M4. Логика bias "спавнить за спиной" конфликтует с anti-repeat защитой и усложняет отладку ramp. Оставить для балансировочного pass после первого playthrough.

## §2 Distance Constraints

**Минимальная дистанция spawn-point → игрок: 12 units.**

Обоснование через координаты: P(0,1,+10), S1(−2,0,+19) — дистанция ~9.2 units. S1 **ниже** порога 12. Это намеренно: S1 — самый опасный spawn именно потому что близко и за спиной. Ограничение применяется только если игрок активно находится вплотную к spawn-point (то есть отошёл к южной стене). В нормальном движении P находится в центре/северной зоне, и S1 снова открывается.

**Алгоритм проверки:**
1. Выбрать точку по weighted random.
2. Проверить дистанцию point → player.
3. Если < 12 units — пропустить, взять следующую по весу.
4. Если **все 4 точки** < 12 units (игрок в центре арены при нестандартном layout или edge-case) — **спавнить на самой дальней точке без skip**. Ramp не прерывается. Обоснование: арена 40×40 с P в центре даёт максимальную дистанцию до S2 ≈ 29 units, до S3 ≈ 22 units — кейс "все < 12" практически невозможен, но fallback нужен.

**Максимальная дистанция — не ограничивать.** Нет смысла запрещать дальние спавны: стрелок с S2 в 29 units — именно та угроза, которую layout проектировал.

## §3 Per-Type Spawn-Point Routing

**Тип и точка — ортогональные параметры. Routing по типу не вводить в M4.**

Почему отклонён жёсткий routing (стрелок = дальняя точка, ближнебой = ближняя):
- Детерминизм убивает сюрприз: через 5 минут игрок знает "S1 = всегда ближнебой, S2 = всегда стрелок" и строит механический ответ, не ситуационный.
- Противоречит anti-kластеризации из M3_identity §4: стрелки не должны кластеризоваться с ближнебоями — это задача AI поведения (perimeter-держание), не spawn routing.
- Ближнебой из S2 (далёкий север, 29 units) — интересная ситуация: игрок видит rush издалека, имеет время на dash + pre-aim. Это разнообразие, не баг.

**Единственное soft-правило:** при spawn stрелка — проверить, что в радиусе 8 units от выбранной точки нет другого стрелка-в-movement (perimeter-держание). Если есть — reroll type на ближнебоя для этого тика. Это минимальная anti-кластеризация без жёсткого routing.

**Type ratio по времени run'а (design intent для systems-designer):**
- 0–3 мин: 70% ближнебой / 30% стрелок — учим игрока Kill verb, hook становится понятным через физическое давление.
- 3–8 мин: 50/50 — вводим интеллектуальный слой "кого убить первым".
- 8–15 мин: 40% ближнебой / 60% стрелок — доминирует позиционный налог, cap drain накапливается быстрее. Финальный pressure phase.

## §4 Point Cooldown

**Cooldown на точку: 3 секунды.**

Обоснование: ramp formula `interval = 4 / (1 + time*0.015)` даёт на старте ~4 сек между спавнами, к 10 мин ~1.9 сек. При 4 точках и cooldown 3 сек конфликт наступает только после ~7 минут когда interval падает ниже 3/4 ≈ 0.75 сек — это поздний game, давление оправдано.

**При коллизии (все 4 точки на cooldown):**
Спавн задерживается до первой освободившейся точки. Ramp-таймер **не сбрасывается** — он продолжает тикать. Следующий спавн происходит при первом освобождении точки. Это значит: если interval = 1.5 сек, а cooldown заблокировал все 4 точки на 2 сек — следующие 2 тика "съедаются" в один. Результат: очередь не накапливается, но ramp не теряет темп. Систем-дизайнеру: проверить что очередь не растёт бесконечно — нужен hard cap на ожидающих спавн (например max 2 в очереди, остальные дропаются).

**Cooldown не сбрасывается при смерти врага** — только по времени. Это важно: не создаём incentive быстро убивать ради освобождения точки, игрок не должен думать о точках вообще.

## §5 Spawn Telegraph

**Модель: delayed visual с instant audio. Не instant Devil Daggers стиль.**

Devil Daggers (instant shadow из земли) работает для абстрактного хоррора. VGate — sci-fi, читаемость через silhouette критична (M3_identity §3B). Instant spawn без telegraph = смерти которые игрок не понимает = баг уровня по принципу дизайна.

**Последовательность:**
1. **Frame 0**: spawn-point мигает/светится (короткий particle burst или emission pulse на геометрии точки). Звук немедленно: тип-специфичный.
2. **Frame 0–250 мс**: enemy materializes с alpha fade-in (0 → 1 за 250 мс).
3. **Frame 250 мс**: враг полностью visible, начинает AI behaviour.

**Звук по типам:**
- Ближнебой: низкочастотный thud/clank — слышен через стены (как в M3_identity §3C "слышно за 5-7 units через cover"). S1 за спиной игрока = звук ближнебоя из-за спины = critical early warning.
- Стрелок: высокочастотный whine/hum — тот же звук что при pre-attack charge (M3_identity §1, Step 1), но тише. Игрок выучивает этот звук как "стрелок появился или готовится стрелять".

**250 мс — минимальный window.** Достаточно для поворота и принятия решения "бежать или убивать", недостаточно для комфортного прицеливания. Tension сохраняется.

**Визуальный сигнал на точке не требует отдельного UI-маркера** — достаточно particle/emission на самой геометрии арены. Если в M4 нет particle system — простой Color flash на Marker3D материале через tween (250 мс). Feel-engineer pass на M5 уточняет.

## §6 Overflow Protection

**Cap на живых врагов: 20 (M4 старт). Ramp при достижении cap — пауза тиков.**

Обоснование cap 20 (не 30 из брифа): 30 живых врагов при арене 40×40 = 0.075 врагов/unit². NavigationAgent3D начнёт conflicts, визуальный chaos убивает читаемость. Exit criteria PLAN.md §3 п.6 — "60 FPS стабильно при 30+ врагах" — это технический ceiling, не дизайновый target. Дизайновый cap = порог где игрок теряет agency, не где падают FPS. При 20 врагах и 3 cover-блоках арена уже читается как хаос на позднем ramp — это правильное давление.

**При достижении cap:**
- Ramp-таймер **паузится** (не сбрасывается, не накапливает очередь).
- Возобновляется при первом kill — с текущего значения interval'а, не с нуля.
- Это создаёт прямую связь "kill = разблокировка ramp" — Kill verb получает дополнительный слой смысла помимо cap restore.

**Soft warning для систем-дизайнера:** если игрок доживает до cap 20 чаще чем ~5-7 мин в забеге — cap, возможно, слишком высокий или kill restore слишком эффективен. Playtest M4 покажет.

## Summary

| Параметр | Решение |
|---|---|
| Selection | Weighted random: S1=3, S2=3, S3=2, S4=2; anti-repeat (no consecutive same point) |
| Min distance | 12 units; fallback = самая дальняя точка |
| Type routing | Ортогонально от point selection; soft anti-кластеризация стрелков (8 units check) |
| Type ratio | 70/30 melee → 50/50 → 40/60 ranged по фазам run'а |
| Point cooldown | 3 сек; при коллизии — ждать освобождения, ramp не сбрасывается |
| Telegraph | 250 мс alpha fade-in + instant type-specific audio на frame 0 |
| Overflow cap | 20 живых; ramp паузится, возобновляется на первом kill |

## Флаг: достаточность 4 точек для 10-15 мин

4 точки достаточны для M4. Аргумент: variety дают тип + позиция игрока + anti-repeat, не количество точек. Devil Daggers работает с фиксированным кольцом — разнообразие через escalation density, не через layout.

**Потенциальная проблема к позднему ramp (8–15 мин):** когда interval падает до 1–2 сек, а на арене 15+ врагов, игрок перестаёт замечать откуда конкретно спавнятся новые враги — spatial awareness теряется в chaos. Это ожидаемо для Devil Daggers формы (chaos = финальная фаза давления), но если exit criteria PLAN.md п.4 "ощущение нарастающего давления" должно сохранять читаемость, а не просто chaos — флагнуть на playtest M4 и рассмотреть 5-ю точку в NE-квадранте (x=+18, z=−15). Не добавлять до плейтеста.
